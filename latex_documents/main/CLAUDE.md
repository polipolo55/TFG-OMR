# CLAUDE.md — the TFG memoir (`latex_documents/main/`)

This directory is the LaTeX source of the thesis memoir for **Optical Music Recognition for
Jazz Lead Sheets**, a Bachelor's thesis (TFG) at FIB/UPC, Grau en Enginyeria Informàtica
(Computació), by Pol Casanovas Puig. The code it describes lives at the repository root; see
the root `CLAUDE.md` for the software side.

## Hard Constraints — Never Violate These

**1. Never build the PDF.** Do not run `latexmk`, `pdflatex`, `lualatex`, `biber`, or `make`.
The user compiles `main.pdf` themselves. Your job ends at editing `.tex`/`.bib` source. If you
need to check the output of a change, reason about the source or ask the user to compile — do
not invoke the toolchain.

**2. The body is English; only the three abstracts switch language.**
Art. 16 requires one consistent body language. `00_abstracts.tex` uses
`\selectlanguage{catalan}` / `{spanish}` / `{english}` for the *Resum / Resumen / Abstract*
and nothing else does. Never introduce Catalan or Spanish prose into a body chapter.

**3. Package load order in `main.tex` is deliberate — preserve it.**
`biblatex` loads *before* `tfg.sty`; `tfg.sty` loads `hyperref`; `cleveref` loads *after*
`tfg.sty` (i.e. after `hyperref`). Reordering these breaks references or citations. The
comments in `main.tex` state each "must come before/after" relationship — keep them true.

**4. Cross-references use `cleveref` in prose.**
Write `\Cref{ch:design}` at sentence start, `\cref{fig:foo}` mid-sentence. `cleveref` is loaded
`[capitalize, nameinlink]`, so it emits "Chapter 4", "Figure 7" with the link on the word.
`Fig.~\ref{}` / `Chapter~\ref{}` (a manual prefix word glued to a raw `\ref`) is the defect to
avoid. The **one sanctioned exception** is the compact section pointer `\S\ref{sec:...}` ("§ 4.2"),
used inside dense table cells such as the decision register in `04_design.tex`, where a full
`\Cref{}` would bloat the column. Use `\S\ref` only in that table-cell context; everywhere in
prose, use `\Cref`/`\cref`.

**5. `tfg.sty` mirrors `src/style.py`.** The colour palette (`tfgPrimary` … `tfgNeutralLight`)
is the LaTeX twin of the `PALETTE` in `src/style.py`, so figures generated by the code match
the document. If you change a colour here, the matplotlib palette is the source of truth —
sync, don't diverge.

**6. Citations are numeric IEEE in citation order.** `biblatex` is configured
`[style=ieee, sorting=none]`. Every prior-work mention carries a `\cite{key}` (or `\textcite`
when the author name is part of the sentence). Author-name-without-`\cite` is a defect.

## Build (for reference only — you do not run it)

The toolchain is **latexmk → pdflatex + biber**, IEEE biblatex bibliography, `svg` package
(needs Inkscape on PATH for any `\includesvg`). Build artifacts (`main.aux`, `.bcf`, `.bbl`,
`.fdb_latexmk`, `.log`, `main.pdf`, …) are present in this directory and are regenerated by
the user — never hand-edit them, and don't treat a stale `.aux` as ground truth for whether a
label resolves.

## Conventions Actually in Use

**Custom macros (from `tfg.sty`):**

- `\term{...}` — italic, for the *first* use of a domain term (used ~59×). After first use,
  switch to the plain word or the acronym.
- `\code{...}` — monospace for identifiers, filenames, tokens, CLI flags (used ~400×). Use this
  for anything code-ish, not `\texttt` directly.
- `\important{...}` — bold highlight colour; defined but reserve for genuine emphasis.
- `\todo{...}` — renders an orange `[TODO: …]` marker. **There are unfinished `\todo` markers
  in the source** — grep before claiming a chapter is complete (see Validation). Orange is
  reserved for `\todo` and unresolved work; don't use `tfgSecondary` for normal emphasis.

**Tables:** `booktabs` (`\toprule`/`\midrule`/`\bottomrule`), plus helpers `\tfgheadrule`
(coloured top rule), `\tfgrowsep` (light row separator), and the `L` column type
(`\raggedright` wrapping `tabularx` column). All tables get a `tab:` label and a full-sentence
caption. New tables follow this style — no vertical rules, no `\hline`.

**Figures:**

- Raster/vector: `\includegraphics[width=...\textwidth]{figures/fig_name}` — **no file
  extension** (graphicx picks `.pdf`/`.png`). Widths in the source range `0.55`–`1.0\textwidth`.
- TikZ: the `figure` environment, caption, and `\label` live *inside* `figures/fig_name.tex`;
  the chapter pulls it with `\input{figures/fig_name}` (no extension, no `figure` wrapper at the
  call site). TikZ libraries are loaded once in `main.tex` — don't re-`\usetikzlibrary` in a
  figure file.
- Captions are full sentences ending in a period (`caption` is set `labelsep=period`). Every
  figure/table is `\cref`-ed in the text *before* it appears.

**Numbers & units:** `siunitx` is loaded — use `\SI{3}{\percent}`, `\SI{12}{GB}` in the
English body. In the **Catalan/Spanish abstracts only**, use a comma decimal (`1{,}19\%`) to
match locale; the English abstract and body use a period (`1.19\%`). Thin-space thousands:
`4\,604`. Flats/sharps in prose: `B\(\flat\)`, `E\(\flat\)`.

**Quotes:** `csquotes` is loaded — use `\enquote{...}` rather than raw  ''``.

## Label Scheme

Labels are prefixed by kind. Match the scheme when adding new ones:

| Prefix   | Kind               | Examples                                                                                                              |
| -------- | ------------------ | --------------------------------------------------------------------------------------------------------------------- |
| `ch:`  | chapter            | `ch:intro`, `ch:soa`, `ch:mgmt`, `ch:design`, `ch:impl`, `ch:results`, `ch:sustain`, `ch:conclusions` |
| `sec:` | section/subsection | `sec:design-model`, `sec:mgmt-budget`, `sec:conc-objectives`                                                    |
| `fig:` | figure             | `fig:training-curves`, `fig:ser-distribution`, `fig:lmx-example`                                                |
| `tab:` | table              | `tab:results-headline`, `tab:hyperparams-omr`, `tab:risk-register`                                              |
| `app:` | appendix           | `app:notation-primer`, `app:hyperparams`, `app:qual`                                                            |

Design's "Alternatives considered" subsections use the `-alts` suffix
(`sec:design-model-alts`, `sec:design-pipeline-alts`, …) — keep that convention when adding a
new design decision, since the rubric checks for one alternatives analysis per major decision.

## Chapter ↔ Rubric (summary — details in the `writing-tfg-fib` skill)

`01` Intro/objectives · `02` State of the Art · `03` Management · `04` Design (*why*) ·
`05` Implementation (*how*) · `06` Results/evaluation · `07` Sustainability (mandatory, three
dimensions + SDGs) · `08` Conclusions (objectives-revisited + competences map, 4–6 pp).
The "*why* in Design, *how* in Implementation" split is the most common boundary question —
when content could go in either, put justification in Ch. 4 and engineering detail in Ch. 5.

## Common Pitfalls (specific to this source)

- **Quoting the numbers.** Headline metrics (SER 0.28% clean / 0.23% scan-simulated, melodic
  SER 0.02% clean / 0.03% scan, perfect-transcription 92.1% clean / 94.1% scan, 4 608-sample
  test split) appear in the abstract, intro, results, and conclusions. If a result changes, it
  must change in **all** places — grep the figure before editing one occurrence.
- **Objectives are measurable and are revisited.** Ch. 1 states numeric targets (aggregate SER
  ≤ 3% scan-simulated, ≤ 2% clean, ≥ 30% relative reduction on fine-tuned real pages). Ch. 8
  must tick each off as met/partial/dropped. Don't soften a Ch. 1 target without updating Ch. 8.
- **Editing a TikZ figure.** Its caption/label are in the `figures/*.tex` file, not the chapter
  — edit the right file, and remember the chapter only `\input`s it.
- **New citation key.** Add it to `references.bib` with the camelCase `authorTitleYear`
  convention; numbering is automatic (citation order, IEEE). Don't hand-number references.
- **Stale `.aux`/`.bbl`.** These reflect the last user build, not your edits. A missing label
  won't error until the user recompiles — verify label existence in the `.tex` source, not the
  log.
