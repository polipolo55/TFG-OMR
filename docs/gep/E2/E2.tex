\section{Project Overview}
\label{sec:overview}

This document presents the temporal planning for the project \emph{Optical Music Recognition for Jazz Lead Sheets: A CRNN-CTC Approach for Monophonic Lines}. The planning follows the agile, iterative methodology described in Deliverable~1, organised into five sprints of $\sim$4 weeks each plus a concurrent project management work package (WP0). Sprints~3 and~4 form a \textbf{refinement loop}: after initial training and evaluation (Sprint~3), error analysis feeds back into data pipeline and model improvements (Sprint~4).

The project runs from February to June 2026 (21 weeks), with a total estimated effort of \textbf{509 hours} at 4--5\,h/day. It is carried out by a single developer who fulfils all roles, with biweekly advisory meetings ($\sim$1.5\,h each) with the thesis supervisor. \textbf{Common resources} for all tasks: personal computer; Python~3.14, PyTorch, OpenCV, music21, LilyPond, \LaTeX, Git/GitHub. Task-specific resources are noted below.

% ------------------------------------------
\subsection{WP1 -- Sprint 1: Literature Review \& Baseline (78\,h)}
\label{sec:wp1}

\textit{Weeks 1--4 (February--March). Goal:} establish the theoretical foundation and implement a classical CV baseline.

\begin{description}[style=nextline, nosep, leftmargin=1.2cm]
    \item[T1.1~-~OMR literature survey (25\,h)]
    Review traditional pipelines~\cite{dalitzComparativeStudyStaff2008}, CRNN-CTC~\cite{PrIMuSDataset}, and transformer approaches~\cite{vilaAntoniorv6SMT2026,UfalOlimpicicdar24Practical}. \emph{Dep:} none. \emph{Res:} IEEE/ACM/arXiv access.

    \item[T1.2~-~Dataset exploration (15\,h)]
    Inspect public OMR datasets (PrIMuS, CameraPrimus, MUSCIMA++, DeepScoresV2, OLiMPiC, GrandStaff); assess suitability. \emph{Dep:} T1.1 (partial overlap$^\dagger$).

    \item[T1.3~-~Classical CV baseline (25\,h)]
    Morphological staff-line detection/removal with OpenCV; evaluate on clean, distorted, and scanned images. \emph{Dep:} T1.2.

    \item[T1.4~-~Baseline evaluation (10\,h)]
    Quantitative evaluation (recall, precision, artefact rate) and failure-mode analysis. \emph{Dep:} T1.3.

    \item[T1.5~-~Sprint 1 review (3\,h)]
    Present results to supervisor; confirm Sprint~2 scope. \emph{Dep:} T1.4.
\end{description}

% ------------------------------------------
\subsection{WP2 -- Sprint 2: Dataset Construction (73\,h)}
\label{sec:wp2}

\textit{Weeks 5--8 (March). Goal:} build a synthetic monophonic dataset in Real Book style for CRNN-CTC training.

\begin{description}[style=nextline, nosep, leftmargin=1.2cm]
    \item[T2.1~-~Output encoding design (12\,h)]
    Define LMX-based vocabulary (pitch, octave, duration, rests, accidentals, barlines, clef/key/time signatures, CTC blank). Write PrIMuS-to-LMX conversion. \emph{Dep:} T1.5. \emph{Res:} music21.

    \item[T2.2~-~Synthetic rendering pipeline (35\,h)]
    LilyPond pipeline rendering $\sim$43\,k staff-line images (PNG) in jazz style (LilyJAZZ) with paired LMX annotations. \emph{Dep:} T2.1. \emph{Res:} LilyPond + LilyJAZZ.

    \item[T2.3~-~Data augmentation (15\,h)]
    Gaussian noise/blur, perspective warp to simulate scanned degradation, calibrated against real Real Book scans. \emph{Dep:} T2.2. \emph{Res:} OpenCV.

    \item[T2.4~-~Validation and splits (8\,h)]
    Validate image--label alignment, filter degenerate samples, prepare 80/10/10 splits. \emph{Dep:} T2.3.

    \item[T2.5~-~Sprint 2 review (3\,h)]
    Review dataset quality with supervisor. \emph{Dep:} T2.4.
\end{description}

% ------------------------------------------
\subsection{WP3 -- Sprint 3: CRNN-CTC Model Development (75\,h)}
\label{sec:wp3}

\textit{Weeks 9--12 (April). Goal:} design, implement, and perform the initial training run; establish the evaluation framework.

\begin{description}[style=nextline, nosep, leftmargin=1.2cm]
    \item[T3.1~-~CRNN architecture design (25\,h)]
    CNN backbone + bidirectional LSTM + CTC head in PyTorch; lightweight for consumer GPU. \emph{Dep:} T2.5. \emph{Res:} consumer GPU.

    \item[T3.2~-~Training pipeline (20\,h)]
    Training loop (CTC loss, LR scheduler, gradient clipping, early stopping, checkpointing) and data loader with dynamic padding. \emph{Dep:} T3.1.

    \item[T3.3~-~Initial training run (15\,h)]
    First full training run; monitor convergence; document baseline metrics. \emph{Dep:} T3.2. \emph{Res:} GPU.

    \item[T3.4~-~Evaluation framework (12\,h)]
    Greedy CTC decoding, SER computation, per-sample error breakdown, worst-prediction visualisation. Partial overlap with T3.3. \emph{Dep:} T3.2$^\dagger$.

    \item[T3.5~-~Sprint 3 review (3\,h)]
    Analyse results with supervisor; plan Sprint~4 refinement. \emph{Dep:} T3.3, T3.4.
\end{description}

% ------------------------------------------
\subsection{WP4 -- Sprint 4: Iterative Refinement (68\,h)}
\label{sec:wp4}

\textit{Weeks 13--16 (April--May). Goal:} close the train--evaluate--improve loop until SER stabilises. This sprint implements the iterative refinement cycle: after each run, errors are analysed, root causes identified, and targeted fixes applied before retraining.

\begin{description}[style=nextline, nosep, leftmargin=1.2cm]
    \item[T4.1~-~Error analysis (10\,h)]
    Breakdown by error type and token; identify systematic patterns. \emph{Dep:} T3.5.

    \item[T4.2~-~Data pipeline improvements (15\,h)]
    Fix annotation errors, filtering, vocabulary adjustments; re-generate data as needed. \emph{Dep:} T4.1.

    \item[T4.3~-~Model refinement runs (25\,h)]
    Hyperparameter adjustment and iterative retraining until SER stabilises. \emph{Dep:} T4.2. \emph{Res:} GPU.

    \item[T4.4~-~Ablation studies (15\,h)]
    Controlled experiments isolating key design choices; structured ablation table. \emph{Dep:} T4.3. \emph{Res:} GPU.

    \item[T4.5~-~Sprint 4 review (3\,h)]
    Present metrics and ablation results; decide on further iteration vs.\ final evaluation. \emph{Dep:} T4.4.
\end{description}

% ------------------------------------------
\subsection{WP5 -- Sprint 5: Final Evaluation \& Defence (85\,h)}
\label{sec:wp5}

\textit{Weeks 17--21 (May--June). Goal:} definitive evaluation, conditional extension, thesis completion, and defence preparation.

\begin{description}[style=nextline, nosep, leftmargin=1.2cm]
    \item[T5.1~-~Final evaluation (15\,h)]
    Evaluate best model on held-out test set; per-category SER; compare against CV baseline; publication-quality figures. \emph{Dep:} T4.5. \emph{Res:} GPU.

    \item[T5.2~-~Qualitative error analysis (10\,h)]
    Case studies of correct/incorrect predictions; failure-mode analysis by difficulty. \emph{Dep:} T5.1.

    \item[T5.3~-~Extension exploration -- conditional (20\,h)]
    If SER $<$10\% and time permits: prototype chord recognition or page-level staff detection. \textbf{Scope-contingent}; may be reduced or skipped. \emph{Dep:} T5.1. \emph{Res:} GPU.

    \item[T5.4~-~Final thesis writing (30\,h)]
    Complete all chapters, integrate figures/tables/references, proofread. \emph{Dep:} T5.2 (and T5.3 if completed). \emph{Res:} \LaTeX, Zotero.

    \item[T5.5~-~Defence preparation (10\,h)]
    Slides, rehearsal, anticipated questions. \emph{Dep:} T5.4.
\end{description}
