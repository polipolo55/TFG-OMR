\section{Project Overview and Global Data}
\label{sec:overview}

This document presents the temporal planning for the Bachelor's Thesis project \emph{Optical Music Recognition for Jazz Lead Sheets: A CRNN-CTC Approach for Monophonic Lines}. The planning follows an agile, iterative methodology as described in GEP Deliverable~1, with development organised into five sprints plus a concurrent project management work package.

Table~\ref{tab:global-data} summarises the global planning parameters.

The project is carried out by a single developer (the student), with periodic advisory input from the thesis supervisor. Being a solo project, all roles researcher, developer, tester, and technical writer are fulfilled by the same person. The supervisor contributes approximately 1--2 hours per meeting, roughly biweekly.

\subsection{Methodology Recap}

As established in Deliverable~1, the project follows an \textbf{iterative and incremental methodology} inspired by agile principles. Rather than a rigid waterfall sequence, the core technical work (dataset construction, model development, and evaluation) is organised into \textbf{sprints} of approximately four weeks each. Each sprint produces a working, evaluated artefact and concludes with a review that informs the next iteration.

Specifically, Sprints~3 and~4 form an explicit \textbf{refinement loop}: after the initial model is trained and evaluated (Sprint~3), error analysis feeds back into data pipeline improvements and model adjustments (Sprint~4). This cycle can repeat within Sprint~4 as needed before moving to final evaluation.

A concurrent project management work package (WP0) runs throughout the entire project, encompassing documentation, supervisor meetings, and GEP-related deliverables.

\subsection{WP1~-~Sprint 1: Literature Review and Baseline}
\label{sec:wp1}

\textit{Duration: Weeks 1--4 (February -- March 2026).} \\
\textit{Sprint goal:} Establish the theoretical foundation and implement a classical computer vision baseline to quantify the difficulty of the task.

\begin{description}[style=nextline, leftmargin=1.2cm]
    \item[T1.1~-~OMR literature survey (25\,h)]
    Systematic review of OMR approaches: traditional pipelines~\cite{dalitzComparativeStudyStaff2008}, CRNN-CTC methods~\cite{PrIMuSDataset}, and transformer-based systems~\cite{vilaAntoniorv6SMT2026,UfalOlimpicicdar24Practical}. Document findings and annotated bibliography. \\
    \textit{Dependencies:} None (sprint start). \\
    \textit{Resources:} Standard + access to IEEE, ACM, and arXiv repositories.

    \item[T1.2~-~Dataset exploration and analysis (15\,h)]
    Download, inspect, and document publicly available OMR datasets (PrIMuS, CameraPrimus, MUSCIMA++, DeepScoresV2, OLiMPiC, GrandStaff). Analyse encoding formats, annotation quality, and suitability for monophonic training. \\
    \textit{Dependencies:} T1.1 (partial; may overlap).

    \item[T1.3~-~Classical CV baseline implementation (25\,h)]
    Implement a morphological staff-line detection and removal pipeline using OpenCV. Evaluate staff-line recall on clean typeset images (PrIMuS), distorted camera images (CameraPrimus), and handwritten Real Book scans. \\
    \textit{Dependencies:} T1.2.

    \item[T1.4~-~Baseline evaluation and report (10\,h)]
    Quantitative evaluation of the morphological baseline: staff-line recall, precision, noise artefact rate. Document findings and failure modes that motivate the deep learning approach. \\
    \textit{Dependencies:} T1.3.

    \item[T1.5~-~Sprint 1 review (3\,h)]
    Present results to supervisor, discuss lessons learned, and confirm Sprint~2 scope. \\
    \textit{Dependencies:} T1.4.
\end{description}

\textbf{WP1 subtotal: 78\,h}

\subsection{WP2~-~Sprint 2: Dataset Construction}
\label{sec:wp2}

\textit{Duration: Weeks 5--8 (March 2026).} \\
\textit{Sprint goal:} Build a synthetic monophonic dataset in Real Book style with paired image--annotation data suitable for CRNN-CTC training.

\begin{description}[style=nextline, leftmargin=1.2cm]
    \item[T2.1~-~Output encoding design (12\,h)]
    Define the LMX-based symbolic vocabulary: token set (pitch + octave + duration + rests + accidentals + barlines), special tokens (clef, time/key signatures), and the CTC blank token. Write the conversion script from PrIMuS \texttt{.semantic} format to LMX. \\
    \textit{Dependencies:} T1.5. \\
    \textit{Resources:} Standard + music21 library.

    \item[T2.2~-~Synthetic rendering pipeline (35\,h)]
    Develop a LilyPond-based pipeline to render PrIMuS-derived monophonic scores in Real Book jazz style using the LilyJAZZ font. Generate $\sim$43{,}000 staff-line images (PNG) with paired LMX annotations. Handle edge cases: multi-rest tokens, clef filtering, degenerate sequences. \\
    \textit{Dependencies:} T2.1. \\
    \textit{Resources:} Standard + LilyPond with LilyJAZZ package.

    \item[T2.3~-~Data augmentation pipeline (15\,h)]
    Implement augmentation transforms (Gaussian noise, Gaussian blur, perspective warp) to produce a ``scanned'' variant of the dataset that simulates real-world degradation. Calibrate distortion parameters against actual Real Book scan characteristics. \\
    \textit{Dependencies:} T2.2. \\
    \textit{Resources:} Standard + OpenCV.

    \item[T2.4~-~Dataset validation and split preparation (8\,h)]
    Validate image--label alignment, filter corrupted or degenerate samples, and prepare reproducible train/validation/test splits (80/10/10). Produce dataset statistics summary. \\
    \textit{Dependencies:} T2.3.

    \item[T2.5~-~Sprint 2 review (3\,h)]
    Present the dataset to the supervisor, review quality metrics, and confirm readiness for model training. \\
    \textit{Dependencies:} T2.4.
\end{description}

\textbf{WP2 subtotal: 73\,h}


% ------------------------------------------
\subsection{WP3~-~Sprint 3: CRNN-CTC Model Development}
\label{sec:wp3}

\textit{Duration: Weeks 9--12 (April 2026).} \\
\textit{Sprint goal:} Design, implement, and perform an initial training run of the CRNN-CTC model, establishing the evaluation framework.

\begin{description}[style=nextline, leftmargin=1.2cm]
    \item[T3.1~-~CRNN architecture design and implementation (25\,h)]
    Design the CNN backbone (convolutional feature extractor), the bidirectional LSTM sequence module, and the fully connected CTC head. Implement in PyTorch with configurable hyperparameters, keeping the model lightweight enough for training on a consumer GPU. \\
    \textit{Dependencies:} T2.5. \\
    \textit{Resources:} Standard + consumer GPU.

    \item[T3.2~-~Training pipeline implementation (20\,h)]
    Implement the training loop with CTC loss, a learning rate scheduler, gradient clipping, early stopping, and best-model checkpointing. Implement the dataset loader with dynamic padding and collation. \\
    \textit{Dependencies:} T3.1.

    \item[T3.3~-~Initial training run (15\,h)]
    Execute the first full training run. Monitor loss curves, diagnose convergence issues, and document initial baseline metrics. \\
    \textit{Dependencies:} T3.2.

    \item[T3.4~-~Evaluation framework (12\,h)]
    Implement greedy CTC decoding, Symbol Error Rate (SER) computation, per-sample error breakdown (substitutions, insertions, deletions), and worst-prediction visualisation. This task runs in partial overlap with T3.3. \\
    \textit{Dependencies:} T3.2 (may overlap with T3.3).

    \item[T3.5~-~Sprint 3 review (3\,h)]
    Analyse initial training results with the supervisor, identify dominant error modes, and plan the Sprint~4 refinement strategy. \\
    \textit{Dependencies:} T3.3, T3.4.
\end{description}

\textbf{WP3 subtotal: 75\,h}


% ------------------------------------------
\subsection{WP4~-~Sprint 4: Iterative Refinement}
\label{sec:wp4}

\textit{Duration: Weeks 13--16 (April--May 2026).} \\
\textit{Sprint goal:} Close the train--evaluate--improve loop. Quantitatively analyse errors, fix data pipeline issues, refine the model, and retrain until SER stabilises.

This sprint implements the \textbf{iterative refinement cycle} at the core of the agile methodology: after each training run, errors are analysed, root causes identified (data corruption, architecture limitations, filtering gaps), and targeted fixes applied before retraining. This cycle may execute multiple times within the sprint.

\begin{description}[style=nextline, leftmargin=1.2cm]
    \item[T4.1~-~Error analysis of initial run (10\,h)]
    Detailed breakdown of errors by type (deletion, substitution, insertion) and by token. Identify systematic patterns: which symbols are most confused and which image characteristics correlate with high SER. \\
    \textit{Dependencies:} T3.5.

    \item[T4.2~-~Data pipeline improvements (15\,h)]
    Fix data quality issues discovered during error analysis: annotation errors, problematic sample filtering, and vocabulary adjustments. Re-generate or re-filter the dataset as needed. \\
    \textit{Dependencies:} T4.1.

    \item[T4.3~-~Model refinement and retraining (25\,h)]
    Adjust hyperparameters based on error analysis and retrain on the improved dataset. Execute several training runs, each informed by the previous run's error profile, until SER stabilises. \\
    \textit{Dependencies:} T4.2. \\
    \textit{Resources:} Standard + consumer GPU.

    \item[T4.4~-~Ablation studies (15\,h)]
    Conduct controlled experiments to isolate the effect of key design choices in the data pipeline, model architecture, and training configuration. Document results in a structured ablation table. \\
    \textit{Dependencies:} T4.3. \\
    \textit{Resources:} Standard + consumer GPU.

    \item[T4.5~-~Sprint 4 review (3\,h)]
    Present the refined model metrics and ablation results to the supervisor. Decide whether further iteration is needed or whether to proceed to final evaluation. \\
    \textit{Dependencies:} T4.4.
\end{description}

\textbf{WP4 subtotal: 68\,h}


% ------------------------------------------
\subsection{WP5~-~Sprint 5: Final Evaluation, Extension, and Defence}
\label{sec:wp5}

\textit{Duration: Weeks 17--21 (May -- June 2026).} \\
\textit{Sprint goal:} Produce the definitive evaluation, explore conditional extensions, complete the thesis document, and prepare the defence.

\begin{description}[style=nextline, leftmargin=1.2cm]
    \item[T5.1~-~Comprehensive evaluation and benchmarking (15\,h)]
    Evaluate the best model on the held-out test set. Compute aggregate and per-category SER. Compare quantitatively against the classical CV baseline from Sprint~1. Produce publication-quality tables and figures. \\
    \textit{Dependencies:} T4.5. \\
    \textit{Resources:} Standard + consumer GPU.

    \item[T5.2~-~Qualitative error analysis and case studies (10\,h)]
    Select representative correct and incorrect predictions for qualitative discussion. Analyse failure modes by image difficulty (dense passages, unusual symbols, heavily augmented images). Produce annotated visualisations. \\
    \textit{Dependencies:} T5.1.

    \item[T5.3~-~Extension exploration~-~conditional (20\,h)]
    If the monophonic model achieves satisfactory SER ($<$10\%) and time permits: prototype a limited extension, such as chord symbol recognition or page-level staff detection. This task is \textbf{scope-contingent} and may be reduced or skipped without affecting the core deliverables. \\
    \textit{Dependencies:} T5.1. \\
    \textit{Resources:} Standard + consumer GPU.

    \item[T5.4~-~Final thesis writing and revision (30\,h)]
    Complete all thesis chapters: introduction, state of the art, methodology, dataset description, model architecture, experiments and results, conclusions, and future work. Integrate all figures, tables, and references. Perform full proofreading and formatting pass. \\
    \textit{Dependencies:} T5.2 (and T5.3 if completed). \\
    \textit{Resources:} Standard + \LaTeX{} (TeX Live), Zotero.

    \item[T5.5~-~Defence preparation (10\,h)]
    Prepare the presentation slides, rehearse the defence talk, and prepare answers for anticipated questions from the evaluation committee. \\
    \textit{Dependencies:} T5.4.
\end{description}

\textbf{WP5 subtotal: 85\,h}

\section{Risk Management and Contingency Plans}
\label{sec:risks}

This section coordinates with the risks identified in Deliverable~1 (Section~3.6) and defines concrete contingency actions, their impact on the schedule, and the additional resources required.

\subsection{R1~-~Limited Ground-Truth Data}

\textbf{Risk:} Annotated Real Book data is not publicly available. The synthetic dataset may contain errors or lack diversity. \\
\textbf{Likelihood:} High. \quad \textbf{Impact:} High (degraded model accuracy). \\
\textbf{Mitigation (built into plan):} T2.2 and T2.3 dedicate 50 hours to synthetic generation and augmentation, including iterative quality checks (T2.4). An overestimation of 5 hours is embedded in T2.2 to handle rendering edge cases. \\
\textbf{Contingency (Plan B):} If the LilyPond pipeline produces unusable output for $>$10\% of samples, fall back to training exclusively on the original PrIMuS typeset images without jazz-style rendering. This eliminates T2.2 entirely but sacrifices visual domain similarity to the Real Book.
\begin{itemize}[nosep]
    \item \textit{Schedule impact:} Saves $\sim$35\,h from T2.2 if activated early; may require $\sim$10\,h of additional augmentation effort in T2.3 to bridge the domain gap.
    \item \textit{Resources:} No additional resources; PrIMuS data is already downloaded.
\end{itemize}

\subsection{R2~-~Staff Detection Errors Propagating Downstream}

\textbf{Risk:} Incorrectly cropped staff lines feed corrupted input to the CRNN, increasing SER. \\
\textbf{Likelihood:} Medium. \quad \textbf{Impact:} Medium. \\
\textbf{Mitigation (built into plan):} The current pipeline assumes pre-cropped lines (synthetic data is generated at line level). This risk is relevant only for the extension (T5.3) targeting full Real Book pages. \\
\textbf{Contingency (Plan B):} If page-level detection is attempted in T5.3 and fails, restrict the demonstration to manually cropped regions. This does not affect the core deliverables.
\begin{itemize}[nosep]
    \item \textit{Schedule impact:} T5.3 hours reassigned to thesis writing (T5.4).
    \item \textit{Resources:} None additional.
\end{itemize}

\subsection{R3~-~Model Underfitting or CTC Misalignment}

\textbf{Risk:} The CRNN does not converge, or CTC loss plateaus at an unacceptably high SER ($>$30\%). \\
\textbf{Likelihood:} Medium. \quad \textbf{Impact:} High (could compromise the project's central contribution). \\
\textbf{Mitigation (built into plan):} Sprint~4 (68\,h) is entirely dedicated to iterative refinement. T4.1--T4.3 form a systematic analyse--fix--retrain loop. An overestimation of $\sim$5\,h is embedded in T4.3 to allow additional runs. \\
\textbf{Contingency (Plan B):} If after Sprint~4 the model still exceeds 20\% SER, simplify the task: reduce vocabulary to pitch-only (removing durations), train on clean (non-augmented) images, or increase the image height to provide more visual detail. These simplifications can be implemented within T4.2--T4.3 without new tasks.
\begin{itemize}[nosep]
    \item \textit{Schedule impact:} Up to 10\,h from T5.3 (extension) can be reallocated to additional refinement iterations.
    \item \textit{Resources:} Same hardware; longer GPU time per run.
\end{itemize}

\subsection{R4~-~Computational Resource Constraints}

\textbf{Risk:} Training runs take too long on the available hardware, slowing the iteration cycle and delaying Sprint~4. \\
\textbf{Likelihood:} Low--Medium. \quad \textbf{Impact:} Medium (schedule delays). \\
\textbf{Mitigation (built into plan):} The model is designed to be lightweight and uses mixed-precision training to reduce memory usage and training time. \\
\textbf{Contingency (Plan B):} If training time becomes a bottleneck, reduce dataset size, simplify the model architecture, or request temporary access to UPC or cloud GPU resources.
\begin{itemize}[nosep]
    \item \textit{Schedule impact:} Minimal if mitigated early; up to 5\,h overhead for cloud setup.
    \item \textit{Resources:} Potential cloud GPU cost ($\sim$\EUR{}20--50 for short-term rental).
\end{itemize}

\subsection{R5~-~Scope Creep Toward Polyphony}

\textbf{Risk:} Attempting polyphonic extensions prematurely diverts effort from the monophonic core. \\
\textbf{Likelihood:} Medium. \quad \textbf{Impact:} High (incomplete core system). \\
\textbf{Mitigation (built into plan):} Polyphonic extension is isolated in T5.3, explicitly labelled as ``conditional,'' and scheduled \emph{after} final evaluation (T5.1). It cannot start until the monophonic model achieves $<$10\% SER. \\
\textbf{Contingency (Plan B):} If time runs short, T5.3 is reduced to a brief feasibility discussion in the thesis (2--3 hours of writing), and the remaining hours are redirected to T5.4 (thesis completion).
\begin{itemize}[nosep]
    \item \textit{Schedule impact:} Up to 18\,h freed from T5.3 and added to T5.4.
    \item \textit{Resources:} None additional.
\end{itemize}

\subsection{Summary of Schedule Buffers}

The plan includes the following built-in margins to absorb deviations:
\begin{itemize}[nosep]
    \item \textbf{Overestimation in critical tasks:} T2.2 (+5\,h), T4.3 (+5\,h).
    \item \textbf{Conditional extension:} T5.3 (20\,h) serves as a flexible buffer that can be fully reallocated to refinement or writing.
    \item \textbf{Thesis writing throughout:} T0.6 runs concurrently, preventing a documentation bottleneck at the end.
    \item \textbf{Total slack:} Approximately 25--30\,h can be redistributed without exceeding the project end date.
\end{itemize}
